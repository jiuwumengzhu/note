# PYTHON帮助

```python
PS C:\用户\管理员> python -h
用法：C:\Users\Administrator\AppData\Local\Programs\Python\Python39\python.exe [选项] ... [-c cmd | -m 模式 |档案 | -] [参数] ...
选项和参数（以及相应的环境变量）：
-b : 发出关于 str(bytes_instance), str(bytearray_instance) 的警告
         并将字节/字节数组与 str 进行比较。 (-bb: 问题错误)
-B : 不要在导入时写入 .pyc 文件；还有 PYTHONDONTWRITEBYTECODE=x
-c cmd : 程序作为字符串传入（终止选项列表）
-d : 打开解析器调试输出（仅适用于专家，仅适用于
         调试版本）；还有 PYTHONDEBUG=x
-E : 忽略 PYTHON* 环境变量（如 PYTHONPATH）
-h : 打印此帮助信息并退出（还有 --help）
-i : 运行脚本后交互检查；甚至强制提示
         如果标准输入似乎不是终端；还有 PYTHONINSPECT=x
-I : 将 Python 与用户环境隔离（隐含 -E 和 -s）
-m mod ：将库模块作为脚本运行（终止选项列表）
-O : 删除 assert 和 __debug__ 相关的语句；在之前添加 .opt-1
         .pyc 扩展名；还有 PYTHONOPTIMIZE=x
-OO : 做 -O 更改并丢弃文档字符串；添加 .opt-2 之前
         .pyc 扩展名
-q : 在交互式启动时不打印版本和版权信息
-s : 不要将用户站点目录添加到 sys.path;还有 Python 用户网站
-S : 初始化时不暗示“导入站点”
-u ：强制 stdout 和 stderr 流无缓冲；
         此选项对标准输入没有影响；还有 PYTHONUNBUFFERED=x
-v ：详细（跟踪导入语句）；还有 PYTHONVERBOSE=x
         可以多次提供以增加详细程度
-V ：打印 Python 版本号并退出（还有 --version）
         当给出两次时，打印有关构建的更多信息
-W arg : 警告控制； arg 是动作:消息:类别:模块:lineno
         还有 PYTHONWARNINGS=arg
-x : 跳过源代码的第一行，允许使用非 Unix 形式的 #!cmd
-X opt ：设置特定于实现的选项。以下选项可用：

         -X 故障处理程序：启用故障处理程序
         -X oldparser：启用传统的 LL(1) 解析器；也是 PythonOLDPARSER
         -X showrefcount：输出总引用计数和使用次数
             当程序完成时或在程序中的每个语句之后内存块
             交互式解释器。这仅适用于调试版本
         -X tracemalloc：使用以下命令开始跟踪 Python 内存分配
             tracemalloc 模块。默认情况下，只有最近的帧存储在
             一个踪迹的回溯。使用 -X tracemalloc=NFRAME 开始跟踪
             NFRAME 帧的回溯限制
         -X 导入时间：显示每次导入需要多长时间。它显示模块名称，
             累计时间（包括嵌套导入）和自时间（不包括
             嵌套导入）。请注意，它的输出在多线程中可能会损坏
             应用。典型用法是 python3 -X importtime -c 'import asyncio'
         -X dev：启用CPython的“开发模式”，引入额外的运行时
             默认情况下启用成本太高的检查。的影响
             开发者模式：
                * 添加默认警告过滤器，如 -W default
                * 在内存分配器上安装调试钩子：参见 PyMem_SetupDebugHooks() C 函数
                * 启用故障处理模块以在崩溃时转储 Python 回溯
                * 启用异步调试模式
                * 设置 sys.flags 的 dev_mode 属性为 True
                * io.IOBase 析构函数记录 close() 异常
         -X utf8：为操作系统接口启用 UTF-8 模式，覆盖默认值
             区域感知模式。 -X utf8=0 明确禁用 UTF-8 模式（即使它会
             否则自动激活）
         -X pycache_prefix=PATH：允许将 .pyc 文件写入以
             给定目录而不是代码树

--check-hash-based-pycs 总是|默认|从不：
    控制 Python 如何使基于哈希的 .pyc 文件无效
file : 从脚本文件中读取的程序
- : 从 stdin 读取的程序（默认；如果是 tty，则为交互模式）
arg ...: 传递给 sys.argv[1:] 中程序的参数

其他环境变量：
PYTHONSTARTUP：交互式启动时执行的文件（无默认值）
PYTHONPATH : ';' 分隔的目录列表，前缀为
               默认模块搜索路径。结果是 sys.path。
PYTHONHOME ：备用 <prefix> 目录（或 <prefix>;<exec_prefix>）。
               默认模块搜索路径使用 <prefix>\python{major}{minor}。
PYTHONPLATLIBDIR ：覆盖 sys.platlibdir。
PYTHONCASEOK ：忽略“导入”语句中的大小写（Windows）。
PYTHONUTF8：如果设置为 1，则启用 UTF-8 模式。
PYTHONIOENCODING：编码 [:errors] 用于 stdin/stdout/stderr。
PYTHONFAULTHANDLER：转储 Python追溯致命错误。
PYTHONHASHSEED：如果此变量设置为“随机”，则使用随机值
    播种 str 和 bytes 对象的哈希值。 它也可以设置为
    [0,4294967295] 范围内的整数以获取哈希值
    可预测的种子。
PYTHONMALLOC：设置 Python 内存分配器和/或安装调试钩子
    在 Python 内存分配器上。 使用 PYTHONMALLOC=debug 安装调试
    钩子。
PYTHONCOERCECLOCALE：如果此变量设置为 0，则禁用语言环境
    胁迫行为。 使用 PYTHONCOERCECLOCALE=warn 请求显示
    stderr 上的语言环境强制和语言环境兼容性警告。
PYTHONBREAKPOINT：如果此变量设置为 0，则禁用默认值
    调试器。 它可以设置为您选择的调试器的可调用对象。
PYTHONDEVMODE：启用开发模式。
PYTHONPYCACHEPREFIX：字节码缓存 (pyc) 文件的根目录。
```

# JAVA帮助

```java
PS C:\Users\Administrator> java -h
用法: java [-options] class [args...]
           (执行类)
   或  java [-options] -jar jarfile [args...]
           (执行 jar 文件)
其中选项包括:
    -d32          使用 32 位数据模型 (如果可用)
    -d64          使用 64 位数据模型 (如果可用)
    -server       选择 "server" VM
                  默认 VM 是 server.

    -cp <目录和 zip/jar 文件的类搜索路径>
    -classpath <目录和 zip/jar 文件的类搜索路径>
                  用 ; 分隔的目录, JAR 档案
                  和 ZIP 档案列表, 用于搜索类文件。
    -D<名称>=<值>
                  设置系统属性
    -verbose:[class|gc|jni]
                  启用详细输出
    -version      输出产品版本并退出
    -version:<值>
                  警告: 此功能已过时, 将在
                  未来发行版中删除。
                  需要指定的版本才能运行
    -showversion  输出产品版本并继续
    -jre-restrict-search | -no-jre-restrict-search
                  警告: 此功能已过时, 将在
                  未来发行版中删除。
                  在版本搜索中包括/排除用户专用 JRE
    -? -help      输出此帮助消息
    -X            输出非标准选项的帮助
    -ea[:<packagename>...|:<classname>]
    -enableassertions[:<packagename>...|:<classname>]
                  按指定的粒度启用断言
    -da[:<packagename>...|:<classname>]
    -disableassertions[:<packagename>...|:<classname>]
                  禁用具有指定粒度的断言
    -esa | -enablesystemassertions
                  启用系统断言
    -dsa | -disablesystemassertions
                  禁用系统断言
    -agentlib:<libname>[=<选项>]
                  加载本机代理库 <libname>, 例如 -agentlib:hprof
                  另请参阅 -agentlib:jdwp=help 和 -agentlib:hprof=help
    -agentpath:<pathname>[=<选项>]
                  按完整路径名加载本机代理库
    -javaagent:<jarpath>[=<选项>]
                  加载 Java 编程语言代理, 请参阅 java.lang.instrument
    -splash:<imagepath>
                  使用指定的图像显示启动屏幕
```
